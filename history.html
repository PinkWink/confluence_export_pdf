17. 특정 페이지 이미지 깨짐 문제 보고 (2026-03-06)

하나의 노드에서 서로 다른 도메인ID에 접근 페이지를 변환할 때 이미지가 모두 깨지는 문제 보고. 다른 페이지는 정상인데 해당 페이지만 이미지가 PDF/HTML에 포함되지 않음.

Claude Investigation

18. 원인 분석 Root Cause

원인: Confluence Cloud에서 이미지를 인라인 붙여넣기(paste)할 때 간혹 발생하는 버그로, 페이지의 Storage Format에 이미지 파일명이 ri:filename="UNKNOWN_ATTACHMENT"로 저장되어 있었음.

  • 기존 코드는 ri:filename 값으로 첨부파일 API를 조회 → UNKNOWN_ATTACHMENT라는 파일은 존재하지 않아 이미지 다운로드 실패
  • 하지만 ac:alt 속성에는 실제 파일명(예: image-20250307-094951.png)이 정상적으로 포함되어 있었음
  • 첨부파일 API로 확인 결과, 실제 파일들은 모두 정상적으로 서버에 존재
Claude Implementation

19. UNKNOWN_ATTACHMENT fallback 처리 Bug Fix

process_images() 함수에서 ri:filename이 UNKNOWN_ATTACHMENT일 때 ac:alt 속성의 값을 fallback 파일명으로 사용하도록 수정.

  • 수정 위치: confluence_export.py:709-712 (4줄 추가)
  • 로직: ri:filename == "UNKNOWN_ATTACHMENT"이면 같은 매크로의 ac:alt 태그에서 실제 파일명을 추출하여 사용
  • 결과: 해당 페이지의 모든 이미지가 정상적으로 Document PDF, Presentation HTML, Presentation PDF에 포함됨
